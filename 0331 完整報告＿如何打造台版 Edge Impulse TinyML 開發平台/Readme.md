這三篇論文分別探討了嵌入式系統、神經形態運算以及微控制器上的機器學習（TinyML）。以下是根據提供的來源對這三篇論文的概述、遇到的問題及解決方法的詳細整理：

### 1. Bare-Metal RISC-V + NVDLA SoC for Efficient Deep Learning Inference
這篇論文介紹了一種新型的片上系統（SoC）架構，旨在加速邊緣運算應用的深度學習模型推論。

*   **概述：**
    該研究將開源的 **NVIDIA 深度學習加速器 (NVDLA)** 與 Codasip 的 32 位元 **RISC-V 核心 ($\mu RISC-V$)** 緊密結合，並開發出一套產生「裸機（Bare-metal）」組合語言代碼的工具流。該系統在 AMD ZCU102 FPGA 開發板上進行了驗證，並成功執行了 LeNet-5、ResNet-18 和 ResNet-50 等模型。
*   **遇到的問題：**
    1.  **軟體開銷巨大：** 過往研究多依賴 Linux 核心來執行深度學習工作負載，這對資源受限的邊緣設備造成了顯著的效能與存儲開銷。
    2.  **缺乏硬體部署細節：** 許多相關工作僅停留在模擬階段，缺乏在 FPGA 等真實硬體上的部署指導或資源利用率細節。
    3.  **配置生成困難：** NVDLA 官方僅提供基本測試的配置檔，缺乏如何為任意神經網路生成這些配置檔的指南。
*   **解決方法：**
    1.  **裸機執行流：** 開發了一套自動化工具，將 Caffe 模型轉換為 RISC-V 組合語言，直接配置 NVDLA 暫存器，**完全捨棄 Linux 核心**以提升速度與存儲效率。
    2.  **硬體整合優化：** 透過自定義的 NVDLA 封裝器（Wrapper）與 AXI/AHB 橋接器，解決了處理器與加速器介面位元寬度不匹配的問題。
    3.  **配置與權重提取：** 提出一套利用 NVDLA 虛擬平台 (VP) 日誌來提取暫存器交易（CSB）與權重（DBB）的流程，從而支援任意 Caffe 網路。

### 2. AceleradorSNN: A Neuromorphic Cognitive System Integrating Spiking Neural Networks and Dynamic Image Signal Processing on FPGA
這篇論文展示了一個名為 AceleradorSNN 的第三代人工智慧認知系統，專為自主系統（如無人機或自動駕駛）設計。

*   **概述：**
    該架構整合了基於 **脈衝神經網路 (SNN)** 的神經形態處理單元 (NPU)，用於處理動態視覺感測器 (DVS) 的異步數據，並結合了一個可動態重構的 **認知圖像信號處理器 (ISP)** 來優化 RGB 攝像頭輸出。
*   **遇到的問題：**
    1.  **傳統 CNN 的限制：** 傳統卷積神經網路在處理連續幀數據時會產生高運算負載與高能耗。
    2.  **感測器侷限性：** 事件相機（DVS）雖然反應快、功耗低，但缺乏高解析度的紋理與色彩；而傳統 RGB 感測器在高速運動或光線變化劇烈時表現不佳。
    3.  **SNN 訓練困難：** 脈衝發射過程是不可微的，傳統的反向傳播算法難以直接應用。
*   **解決方法：**
    1.  **協同架構：** NPU 作為認知控制器，根據 DVS 偵測到的場景變化（如光線異常），實時調整 ISP 的參數（如曝光、白平衡），達成感測器互補。
    2.  **Surrogate Gradients 訓練：** 使用 **漏電整合觸發 (LIF) 經元模型**，並透過代理梯度（Surrogate Gradients）近似脈衝函數的導數，從而支援標準優化器進行訓練。
    3.  **FPGA 流式處理：** ISP 採用全流水線（Pipelined）設計，逐個處理像素而不需要存儲完整圖像幀，大幅節省硬體面積。

### 3. TinyOL: TinyML with Online-Learning on Microcontrollers
這篇論文提出了一種名為 **TinyOL** 的新技術，旨在讓資源極其受限的微控制器 (MCU) 具備「在線學習（Online Learning）」的能力。

*   **概述：**
    TinyOL 是一個基於 C++ 的系統，可以作為額外的一層附加到現有的預訓練神經網路上。它支援在 MCU 上直接使用流式數據進行**增量式訓練**，而無需重新燒錄或依賴雲端。
*   **遇到的問題：**
    1.  **模型靜態化：** 目前的 TinyML 多為「離線訓練、在線推論」，一旦環境發生改變（概念漂移，Concept Drift），固定的模型效能就會大幅下降。
    2.  **資源受限：** MCU 的記憶體（RAM）與運算能力極小（如 256KB RAM），難以執行傳統的批次（Batch）訓練。
    3.  **設備差異性：** 即使是同型號的機器也會有細微物理差異，通用的預訓練模型難以完全適配每台設備。
*   **解決方法：**
    1.  **在線增量學習：** 透過**在線隨機梯度下降 (SGD)**，系統每次僅處理一個數據樣本並立即更新權重，隨後即可丟棄該數據，極大節省了存儲空間。
    2.  **動態層結構：** TinyOL 允許在運行時修改輸出層結構。例如，當偵測到新的異常類別時，可以動態增加神經元來學習新類別。
    3.  **遷移學習概念：** 將預訓練模型視為固定的特徵提取器（Frozen Graph），僅對 TinyOL 附加的最後一層進行動態訓練，平衡了運算量與適配能力。

## 心得
這次歐尼克斯許哲豪博士述說未來“edge ai”的走向,目前以現狀來看,大部分model都運行在pc or mobile上都是因為memory的限制,目前mcu方面需要run model也需要memory,所以只能run一些比較小的model。
目前博士有提到edge比較廣義的稱呼會是無網路環境下跑model,但俠義是要在battery的環境＋無網路才是真正的edge ai,那麼mcu就比較好一點。
大部分人都在擔心未來ai會影響工作走向,但掌握好ai與domain language就不用太擔心這方面的問題。
